P0 durability and acknowledgement validation:
- Invalid provider signatures are rejected before durable persistence.
- Event, Delivery, and Delivery Outbox commit atomically before normal-path 202.
- Duplicate provider events create one Event, one Delivery, and one Outbox row.
- Twenty concurrent retries of the same provider event are deduplicated.
- Transaction failure rolls back the Event rather than leaving partial normal-path state.
- PostgreSQL failure falls back to authenticated encrypted spool and returns 202 only after commit.
- PostgreSQL plus spool failure returns 503.
- Existing Event plus missing Delivery/Outbox is reconciled during spool recovery.
- Redis is not required for provider acknowledgement or API readiness; degradation is reported.

Delivery and network security validation:

- Redirect following is disabled.
- Destination response bodies are bounded and truncated without converting successful 2xx into a retry.
